ca.uqac.lif.cep.io.ReadInputStream Class Reference

Detailed Description

Read contents from a Java InputStream, and outputs its chunks as byte arrays.

It is represented graphically as:

If the processor is instantiated by passing it a File, the processor attempts to auto-detect if it is a "regular" file or a named pipe. In the latter case, the processor will keep trying to fetch bytes from the input stream until the "end of transmission" (EOT) byte is observed.

◆ m_chunkSize

int ca.uqac.lif.cep.io.ReadInputStream.m_chunkSize = 16384
protected

The size of chunks.

The processor will try to read this number of bytes every time it queries the underlying input stream. Setting it to a too small value will cause the reader to loop uselessly to process tiny bits of the string. Setting it to a too large value (i.e. 1 MB) has an equally adverse effect. Experimentally, a sweet spot seems to be 16 kB.
