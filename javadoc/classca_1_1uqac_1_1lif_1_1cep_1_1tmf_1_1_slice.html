Detailed Description

Separates an input trace into different "slices".

The slicer takes as input a processor p and a slicing function f. There exists potentially one instance of p for each value in the image of f.

When an event e arrives, the slicer evaluates c = f(e). This value determines to what instance of p the event will be dispatched. If no slice with such value currently exists, a new instance of p will be created. The processor then outputs a Collection object (typically a set) containing the last events returned by every slice.

The function f may return null, or the special object ToAllSlices. This indicates that no new slice must be created, but that the incoming event must be dispatched to all slices one by one.
